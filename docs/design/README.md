# Design Documentation

> **Estado documental:** carpeta de modelado y diseno. Estos documentos
> preservan Event Storming, DDD estrategico/tactico, UX y decisiones de diseno
> producidas durante el proyecto. No todos son fuente operativa vigente.

## Como leer esta carpeta

- `context-map.md`, `domain-model.md` y documentos de Event Storming son
  referencia de modelado DDD.
- `architecture.md` es historico/superseded por `docs/architecture/`.
- Documentos UX y wireframes describen decisiones de diseno de frontend por
  momento de implementacion; contrastar con el frontend actual y specs SP5.
- Para arquitectura vigente usar `docs/architecture/`.
- Para alcance vigente usar `docs/plans/sp5/PLAN-SP5.md`.
