Horizontes Estratégicos — AtaraxiaDive

Esta vista muestra la organización del proyecto como una secuencia de hipótesis. Cada horizonte responde una pregunta distinta y agrupa subproyectos que producen evidencia concreta.

Del trabajo al aprendizaje

Una WBS tradicional dice qué hay que hacer. Esta vista agrega algo más fuerte: muestra para qué se hizo cada tramo. AtaraxiaDive no avanza solo por entregables; avanza por preguntas que ordenan el experimento, el producto y el material académico.

Lectura completa del recorrido

1. Validar Primero se verifica si el entorno técnico, metodológico y agéntico funciona sin romperse al primer uso real.
2. Construir Luego se expone IEDD + DDD a un dominio real, con reglas, excepciones, datos reales y deuda técnica. Acá se acaba el PowerPoint y empieza la intemperie.
3. Producir Finalmente se busca convertir el conocimiento acumulado en producto operativo, evidencia académica, libro y paper.
