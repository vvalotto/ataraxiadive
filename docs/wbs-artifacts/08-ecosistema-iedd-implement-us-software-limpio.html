Convivencia metodológica en AtaraxiaDive

El experimento no muestra IEDD, implement-us y Software Limpio como piezas separadas. Muestra cómo conviven en un proyecto concreto y se necesitan mutuamente para transformar dominio en software verificable.

Tesis del experimento

AtaraxiaDive funciona como laboratorio donde IEDD define qué debe pensarse, /implement-us organiza cómo se construye y Software Limpio verifica si lo construido conserva calidad y fidelidad arquitectónica.

La gracia no está en usar tres nombres bonitos. La gracia está en la convivencia: especificación, ejecución y verificación dentro del mismo flujo real de trabajo.

Lectura como experimento

Hipótesis 1

Una especificación más clara mejora la calidad del BDD y reduce iteraciones de corrección.

Hipótesis 2

Un skill orquestado permite convertir US en evidencia repetible, no en actividad artesanal opaca.

Hipótesis 3

Los controles de Software Limpio muestran si la implementación respeta modelo, arquitectura y calidad.
